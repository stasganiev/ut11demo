## 5.1. Печать, сохранение в PDF, предпросмотр — OK

Форма `НастройкаБлоков` (строки 65-74) использует полный паттерн БСП:
- `УправлениеПечатьюКлиент.НоваяКоллекцияПечатныхФорм()`
- `ОписаниеПечатнойФормы()` — синоним, табличный документ, имя файла
- `ПараметрыПечати()` — заголовок формы
- `ПечатьДокументов()` — стандартная форма предпросмотра (печать, PDF, Excel, email)

Процедура `Печать()` в ManagerModule (строки 24-44) — корректный паттерн БСП:
`НужноПечататьМакет()` → `ВывестиТабличныйДокументВКоллекцию()`.

## 5.2. Сохранение и загрузка пресетов — OK

| Операция | Механизм |
|---|---|
| Сохранение | Массив `{Идентификатор, Включен}` → `ХранилищеЗначения` → `МенеджерЗаписи.Записать()` |
| Загрузка | `МенеджерЗаписи.Прочитать()` → трёхуровневая валидация → объединение с умолчаниями |
| Новый пользователь | `Выбран() = Ложь` → применяются `НастройкиБлоковПоУмолчанию()` |
| Миграция | `ОбъединитьНастройки()`: удалённые блоки пропускаются, новые — в конец |

## 5.3. Взаимоисключающие блоки — OK

Обработчик `БлокиПечатиВключенПриИзменении` (строки 21-38):
- При включении блока с непустой `Группой` — все блоки той же группы выключаются.
- Группы: `Шапка`, `Товары`, `Подписи`.
- Блоки без группы: `РеквизитыПокупателя`, `ЭтапыОплаты`, `ПримечаниеКДокументу`.

## 5.4. Крайние случаи — OK

| Сценарий | Поведение |
|---|---|
| Только итоги | `ТаблицаТоваровИтоги` использует `ИтогоВсего`/`ИтогоНДС`/`СуммаПрописью` — не зависит от детальной таблицы |
| Только краткая шапка | `ШапкаКраткая` — выводит заголовок, организацию, ИНН; не требует банковских реквизитов |
| Ноль блоков | `АктивныеБлоки.Количество() = 0` → `ПоказатьПредупреждение()` |

## 5.5. Документы с разным составом данных — OK

| Сценарий | Обработка |
|---|---|
| Без НДС | `ЦенаВключаетНДС` управляет расчётом; `СуммаНДС = 0` при необлагаемых операциях |
| Без этапов оплаты | `ВывестиБлокЭтапыОплаты`: проверка `Количество() = 0` → `Возврат` |
| Пустой комментарий | `ВывестиБлокПримечание`: проверка `ПустаяСтрока()` → `Возврат` |

## 5.6. Разделение клиент/сервер — OK

| Модуль | Контекст | Содержимое |
|---|---|---|
| `ManagerModule.bsl` | `#Если Сервер Или ТолстыйКлиент...` | Запросы, формирование ТД, привилегированный режим |
| `ПечатьСчетНаОплатуКонструкторКлиент` | Клиентский общий модуль | Только `ОткрытьФорму()` |
| `НастройкаБлоков/Module.bsl` | `&НаСервере` / `&НаКлиенте` | Бизнес-логика на сервере, UI-обработчики на клиенте |

Серверных вызовов из формы: `СохранитьНастройкиНаСервере()`, `СформироватьНаСервере()`.
Бизнес-логика вынесена в менеджер обработки — не в форме.

## 5.8. Команда в меню печати ЗаказКлиента — OK

Блок (строки 3447-3455 ManagerModule.bsl ЗаказКлиента):

```bsl
// Счёт на оплату (конструктор)
КомандаПечати = КомандыПечати.Добавить();
КомандаПечати.Обработчик    = "ПечатьСчетНаОплатуКонструкторКлиент.ОткрытьКонструктор";
КомандаПечати.МенеджерПечати = "";
КомандаПечати.Идентификатор = "СчетНаОплатуКонструктор";
КомандаПечати.Представление = НСтр("ru = 'Счёт на оплату (конструктор)...'");
КомандаПечати.ПроверкаПроведенияПередПечатью = Истина;
КомандаПечати.Порядок = 3;
```

- Паттерн аналогичен `КомплектДокументовСНастройкой`
- Вставлен перед `ЗаказКлиентаЛокализация.ДобавитьКомандыПечати`
- `Порядок = 3` — после комплектов (1 и 2)
