## 1. Доступ к команде печати

### 1.1. Видимость команды

- [x] Открыть проведённый ЗаказКлиента → меню «Печать» → в списке есть «Счёт на оплату (конструктор)...»
- [x] Команда расположена после «Комплект документов на принтер» и «Комплект документов с настройкой...»
- [x] Открыть список ЗаказовКлиента → выделить проведённый → меню «Печать» → команда доступна
- [x] Выделить несколько проведённых заказов → команда доступна

### 1.2. Проверка проведения

- [x] Открыть непроведённый ЗаказКлиента → нажать «Счёт на оплату (конструктор)...» → система предлагает провести документ (стандартное поведение БСП)
- [x] Отказаться от проведения → форма конструктора не открывается

### 1.3. Открытие формы

- [x] Нажать «Счёт на оплату (конструктор)...» → открывается форма «Настройка блоков»
- [x] Форма открывается без ошибок
- [x] Заголовок формы корректен

### 3.1. Группа «Шапка»

- [x] Включить «Шапка краткая» → «Шапка полная» автоматически выключается

- [x] Выключить обе шапки вручную — разрешено (обе выключены)

### 3.2. Группа «Товары»

- [x] Включить «Таблица товаров (итоги)» → «Таблица товаров (детальная)» автоматически выключается
- [x] Включить обратно «Таблица товаров (детальная)» → «Таблица товаров (итоги)» автоматически выключается

### 3.3. Группа «Подписи»

- [x] Включить обратно «Подписи (полные)» → «Подпись менеджера» автоматически выключается

### 3.4. Блоки без группы

- [x] «Реквизиты покупателя» — включение/выключение не влияет на другие блоки
- [x] «Этапы оплаты» — включение/выключение не влияет на другие блоки
- [x] «Примечание» — включение/выключение не влияет на другие блоки

### 5.1. Сохранение

- [x] Изменить набор включённых блоков и порядок → «Сохранить настройки» → появляется оповещение «Настройки сохранены»
- [x] Повторно открыть конструктор → настройки восстановлены (включённые блоки и порядок)

### 5.2. Независимость по пользователям

- [x] Войти под другим пользователем (с ролью) → открыть конструктор → настройки по умолчанию (не чужие)

### 5.3. Миграционная устойчивость

- [x] Сохранить настройки → обновить конфигурацию (если добавлен/удалён блок) → открыть конструктор → нет ошибки, новые блоки появляются в конце списка, удалённые пропускаются

## 6. Формирование печатной формы — основные сценарии

### 6.1. Стандартный набор (настройки по умолчанию)

- [x] Нажать «Сформировать» → открывается стандартная форма предпросмотра БСП
- [x] Печатная форма содержит: полную шапку с банковскими реквизитами, реквизиты покупателя, детальную таблицу товаров, полные подписи
- [x] Заголовок предпросмотра: «Счёт на оплату (конструктор)»
- [x] Данные организации корректны (наименование, ИНН, КПП, юр. адрес, банк, БИК, р/с, к/с)
- [x] Данные контрагента корректны (наименование, ИНН, КПП, адрес)
- [x] Номер и дата документа корректны

### 6.2. Таблица товаров (детальная)

- [x] Все товары из заказа отображаются (кроме отменённых строк)
- [x] Колонки: №, номенклатура, ед.изм., количество, цена, сумма, ставка НДС, сумма НДС, всего
- [x] Нумерация строк последовательная (1, 2, 3...)
- [x] Подвал: итого сумма, итого НДС, итого всего, сумма прописью
- [x] Сумма прописью соответствует итогу и валюте документа

### 6.3. Подписи

- [x] «Подписи (полные)» — отображается руководитель и главный бухгалтер
- [x] «Подпись менеджера» — отображается ФИО менеджера в формате «Фамилия И.О.»

## 7. Формирование — вариации данных

### 7.1. НДС

- [x] Заказ с НДС (цена включает НДС) → суммы корректны, «Всего» = «Сумма»
- [x] Заказ с НДС (цена не включает НДС) → «Всего» = «Сумма» + «НДС»
- [x] Заказ без НДС (ПродажаНеОблагаетсяНДС) → столбец НДС = 0, итого НДС = 0

### 7.2. Этапы оплаты

- [x] Включить блок «Этапы оплаты» для заказа с графиком оплаты → таблица этапов отображается (этап, дата, сумма, %)
- [x] Заказ без этапов оплаты → блок «Этапы оплаты» включён, но таблица не выводится (пустой блок пропускается)

### 7.3. Комментарий

- [x] Включить блок «Примечание» для заказа с заполненным `ДополнительнаяИнформация` → примечание отображается
- [x] Заказ с пустым `ДополнительнаяИнформация` → блок «Примечание» включён, но ничего не выводится

### 7.4. Номенклатура

- [x] Товар с характеристикой → отображается «Наименование (Характеристика)»
- [x] Товар без характеристики → отображается только наименование
- [x] Товар с заполненным полем «Содержание» → отображается содержание вместо наименования
- [x] Товар с полным наименованием → используется полное наименование (не представление)

### 7.5. Банковские реквизиты

- [x] Организация с банковским счётом → банк, БИК, р/с, к/с из классификатора банков
- [x] Банковский счёт с ручным изменением реквизитов → данные берутся из счёта, а не из классификатора
- [x] Организация без банковского счёта → поля пустые, ошибки нет

## 8. Формирование — крайние случаи

### 8.1. Минимальный набор блоков

- [x] Включить только «Таблица товаров (итоги)» → печатная форма содержит только итоги (всего, НДС, сумма прописью)
- [x] Включить только «Шапка краткая» → печатная форма содержит только краткую шапку

### 8.2. Все блоки включены (по одному из группы)

- [x] Включить: Шапка полная + Реквизиты покупателя + Таблица детальная + Этапы оплаты + Подписи полные + Примечание → все блоки выводятся последовательно

### 8.3. Изменённый порядок

- [x] Переместить «Примечание» перед «Таблицей товаров» → в печатной форме примечание выше таблицы
- [x] Переместить «Реквизиты покупателя» в конец → в печатной форме реквизиты после подписей

### 8.4. Ноль блоков

- [x] Снять все флажки → «Сформировать» → предупреждение «Не выбрано ни одного блока для печати.»
